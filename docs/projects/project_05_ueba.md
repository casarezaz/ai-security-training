## Overview

Build a UEBA system that learns baseline behavioral profiles for users and devices, then detects anomalous activity indicative of compromised accounts, insider threats, or lateral movement. The AI engine behind modern SIEM platforms.

## Why This Convergence Matters

UEBA spots when legitimate credentials are used in illegitimate ways. This catches compromised accounts and insider threats that signature-based systems miss entirely.

## Learning Objectives

- Behavioral profiling with statistical baselines and ML
- Anomaly detection across multiple behavioral dimensions
- Risk scoring and temporal pattern analysis
- Alert correlation and event threading

## Implementation Steps

1. Generate or obtain user activity logs dataset (LANL cyber dataset or synthetic)
2. Build per-user behavioral baselines from historical activity
3. Implement multiple anomaly detectors: Isolation Forest, LSTM autoencoder, CUSUM
4. Build peer group analysis module to detect outliers within user populations
5. Implement risk score aggregation across dimensions
6. Build alert correlation engine for event threading
7. Create investigation dashboards with historical context
8. Implement feedback loops to tune detectors

## Deliverables

- Per-user behavioral profiling engine
- Multi-dimensional anomaly detection system
- Risk score aggregation with alert correlation
- Investigation dashboard with analyst feedback loops
