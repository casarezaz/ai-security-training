## Overview

Build a comprehensive testing and defense framework for LLM applications. Implement prompt injection attacks, test for data leakage, build guardrails.

## Why This Convergence Matters

Prompt injection is to LLMs what SQL injection was to web apps in 2005. As organizations deploy LLMs in production, understanding and defending against these attacks is critical.

## Learning Objectives

- Prompt injection taxonomy and attack vectors
- Automated red-teaming techniques for LLMs
- Input/output guardrails and filtering
- Training data extraction and privacy testing

## Implementation Steps

1. Set up target LLM application with system prompt, RAG, and tool-calling
2. Implement direct prompt injection attacks
3. Build indirect injection attacks via RAG/document poisoning
4. Test for training data extraction (membership inference)
5. Implement system prompt extraction techniques
6. Build input guardrails (regex + fine-tuned BERT classifier)
7. Build output guardrails for sensitive information filtering
8. Implement layered defense architecture
9. Create red team report with findings and recommendations

## Deliverables

- Automated LLM red-teaming toolkit
- Input/output guardrail system
- Layered defense architecture
- Red team findings report with remediation guidance
