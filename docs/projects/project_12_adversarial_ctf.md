## Overview

Design and build a CTF platform focused on adversarial ML challenges. Players attack and defend ML models across multiple challenge categories.

## Why This Convergence Matters

CTFs are the fastest way to build and demonstrate practical security skills. An adversarial ML CTF is novel and showcases deep expertise in both AI and security.

## Learning Objectives

- Design challenge categories with escalating difficulty
- Implement automated scoring and validation systems
- Build isolated and secure challenge environments
- Create realistic attack scenarios

## Implementation Steps

1. Design challenge categories with escalating difficulty (evasion, extraction, defense, LLM attacks)
2. Build challenge infrastructure with containerized environments (Docker)
3. Implement 3-4 evasion challenges (bypass ML detectors)
4. Implement 2-3 extraction challenges (steal models, extract data)
5. Implement 2-3 LLM challenges (prompt injection, jailbreaking)
6. Implement 2-3 defense challenges (build robust models)
7. Create dynamic scoreboard with team rankings
8. Write comprehensive solution guides and writeups

## Deliverables

- Complete CTF platform with 10+ challenges across 5 categories
- Containerized isolated environments for each challenge
- Automated scoring and flag validation system
- Dynamic scoreboard with team rankings and progress tracking
- Solution guides and technical writeups for all challenges
- Infrastructure documentation for deployment

## Career Impact

Deep mastery demonstration. Host at DEF CON AI Village or BSides for visibility ($170-230K+ opportunities).
